# Homepage / section planning

Goal: produce a concrete, ordered build spec — not vague advice.

Steps:
1. State the one decision a visitor should make on this page (usually: "worth interviewing").
2. Propose the section order and, for each section: its single job, the most important thing it says, and the proof it shows.
3. Apply the portfolio rules: skills/projects/strengths lead; no resume timeline; every section justified or cut.
4. Define featured-project selection (depth over breadth; problem → approach → impact).
5. List clichés to avoid for this page.
6. Output a numbered spec ready for the ui-designer and frontend-engineer, plus the content slots the content-editor needs to fill.

For deeper positioning work, delegate to the `portfolio-strategist` agent.
